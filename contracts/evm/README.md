# StellaX EVM Bridge

This directory is part of StellaX, not a disposable artifact. It contains the EVM-side Axelar bridge used for EVM ↔ Stellar collateral movement.

## What belongs in source control

- `src/StellaXBridgeEVM.sol` — the bridge contract deployed to EVM chains.
- `script/Deploy.s.sol` — Avalanche Fuji deployment script.
- `script/DeployMultiChain.s.sol` — Arbitrum/Base/Optimism/Avalanche deployment script.
- `test/*.t.sol` — Foundry unit and fork tests.
- `foundry.toml` — Foundry configuration.
- `lib/forge-std/` — local Foundry test/script dependency if this repo vendors dependencies.

## What should stay ignored

- `out/` — compiler artifacts and ABIs generated by `forge build` or `forge test`.
- `cache/` — Foundry compile/test cache.
- `broadcast/` — deployment transaction logs generated by `forge script --broadcast`.

These generated directories are ignored at the repository root.

## Commands

From the repository root:

```bash
pnpm evm:build
pnpm evm:test
```

Or from this directory:

```bash
forge build
forge test
```

## Testnet role

The Fuji deployment path is used by the frontend deposit flow and the Stellar bridge keeper:

1. User deposits aUSDC into `StellaXBridgeEVM.depositToStellar()` on Avalanche Fuji.
2. Axelar relays the GMP message to the Stellar bridge.
3. The bridge keeper observes the Axelar GMP event and credits StellaX collateral on Stellar.
